📖 คู่มือการใช้งานระบบ LPHO BMD
Loei Provincial Public Health Office Budget Management Dashboard
(Version: 690416-612-77)

ระบบบริหารจัดการงบประมาณและติดตามแผนปฏิบัติการประจำปี สำนักงานสาธารณสุขจังหวัดเลย ถูกออกแบบมาให้ใช้งานง่าย รวดเร็ว และประมวลผลข้อมูลแบบ Real-time เพื่อช่วยลดภาระงานด้านเอกสารและเพิ่มประสิทธิภาพในการบริหารงบประมาณ

🔐 1. การเข้าสู่ระบบและสิทธิ์การใช้งาน (Login & Roles)
ระบบมีความปลอดภัยสูงด้วยการเข้าสู่ระบบผ่าน "อีเมล" โดยจะแบ่งสิทธิ์การใช้งานออกเป็น 4 ระดับ ดังนี้:
- ADMIN (ผู้ดูแลระบบ): เข้าถึงได้ทุกเมนู สามารถบันทึก แก้ไข ลบรายการ นำเข้าแผนงาน และจัดการสิทธิ์ผู้ใช้งานท่านอื่นได้
- EDITOR (เจ้าหน้าที่การเงิน/ผู้บันทึกข้อมูล): สามารถดูข้อมูล บันทึกรายการเบิกจ่าย และบันทึกรายการเงินยืมได้ แต่ไม่สามารถแก้ไขแผนงานหลักหรือจัดการผู้ใช้งานได้
- VIEWER (ผู้บริหาร/หัวหน้ากลุ่มงาน): สามารถดูข้อมูลกราฟ แดชบอร์ด และรายงานสรุปต่างๆ ได้อย่างเดียว ไม่สามารถบันทึกหรือแก้ไขข้อมูลใดๆ ในระบบได้
- GUEST (ผู้เยี่ยมชม): ไม่สามารถเข้าถึงข้อมูลเชิงลึกได้จนกว่าจะได้รับการอนุมัติสิทธิ์จาก Admin
(หมายเหตุ: ระบบมีฟังก์ชันความปลอดภัย หากไม่มีการขยับเมาส์หรือใช้งานเกิน 15 นาที ระบบจะทำการออกจากระบบอัตโนมัติ)

📊 2. เมนูและการใช้งานหลัก (Main Features)
2.1 หน้าแดชบอร์ด & รายงานสรุป (Dashboard & Summary)
- แสดงภาพรวมงบประมาณ (จัดสรร, เบิกจ่าย, คงเหลือ) ผ่านกราฟวงกลมและแท่งที่ดูเข้าใจง่าย
- สามารถเลือกดูข้อมูลแบบแยกตาม "ประเภทงบ" หรือ "แหล่งงบประมาณ" ได้
- ทริคการใช้งาน: ในหน้ารายงานสรุป สามารถคลิกปุ่ม "ดู" ท้ายชื่อกลุ่มงาน เพื่อเปิดดูรายละเอียดโครงการย่อย (Tree View) ที่ซ่อนอยู่ได้ทันที

2.2 หน้าจัดสรรงบประมาณ (Master Plan)
- นำเข้าข้อมูล: Admin สามารถนำเข้าไฟล์ Excel โครงการเริ่มต้นประจำปีเข้าระบบได้ทีละมากๆ
- บันทึกจัดสรรงบ: ใช้สำหรับบันทึกการเติมเงินเข้าโครงการ หรืออัปเดตยอดงบประมาณตามหนังสือสั่งการ

2.3 หน้าบันทึกเบิกจ่าย (Transaction)
- ใช้บันทึกการตัดจ่ายเงินจริงออกจากโครงการ
- ระบบจะคำนวณและหักยอดเงิน "คงเหลือ" ในโครงการให้อัตโนมัติ ป้องกันปัญหาการเบิกจ่ายเงินเกินงบ (งบติดลบ)
- หากมีการบันทึกผิด Admin สามารถกดปุ่มถังขยะสีแดงเพื่อ "ลบรายการ" ซึ่งระบบจะคืนเงินเข้าโครงการให้ทันที

2.4 หน้าระบบเงินยืม (Loan System)
- การยืมเงิน: เมื่อบันทึกยืมเงิน ระบบจะนำเงินไปพักไว้ในช่อง "เงินยืม" และหักยอดจากกระเป๋าคงเหลือ
- การคืนเงิน/ล้างหนี้: กดปุ่ม "คืนเงิน" (ไอคอนมือถือเหรียญ) โดยระบบจะให้กรอก 2 ช่อง คือ "ยอดจ่ายจริงตามบิล" และ "ยอดเงินทอนคืนโครงการ" ระบบจะทำการเคลียร์หนี้และกระจายเงินกลับเข้ากระเป๋าต่างๆ อย่างแม่นยำ

2.5 การส่งออกข้อมูล (Export to Excel)
- ในหน้าจัดสรร เบิกจ่าย และเงินยืม จะมีปุ่ม "ส่งออก Excel" สีเขียว
- ผู้ใช้สามารถเลือกจำนวนรายการที่จะแสดง (เช่น เลือก "ทั้งหมด") แล้วกดปุ่ม ระบบจะโหลดไฟล์ .xlsx ลงเครื่องคอมพิวเตอร์ให้ทันที

📂 ข้อมูลอ้างอิงสำหรับผู้พัฒนา (System Architecture)
โครงสร้างไฟล์ถูกแบ่งหน้าที่ไว้ดังนี้ครับ:
- UI & View (index.html, css.html): โครงสร้างหน้าเว็บหลักและสไตล์ความสวยงาม รวมถึงระบบเมนู Responsive
- Frontend Logic (js.html, masterplan_js.html, auth_js.html): สมองฝั่งหน้าบ้าน จัดการปุ่มกด ตาราง DataTables กราฟ สลับหน้าจอ และรับส่งข้อมูลกับหลังบ้าน
- Backend Logic (code.gs, Auth.gs): สมองฝั่งหลังบ้าน (Google Apps Script) ทำหน้าที่เชื่อมต่อและคำนวณข้อมูลลง Google Sheets โดยตรง (Database)
- Sub-Modules (summary.html, masterplan_ui.html, auth_ui.html): ส่วนประกอบย่อยของหน้าจอที่ถูกแยกไฟล์ไว้เพื่อให้โค้ดอ่านง่ายและไม่ยาวจนเกินไป
